会话技术

会话技术

会话技术可以解决的问题

  • 访问网站走的是HTTP协议，协议具有无连接无状态的特性，对事物的处理不具有记忆能力
  • 在web项目中，很多时候需要将会话数据记忆存储，以便后续的程序能够随时调用和使用

会话技术是什么

  • 在web访问过程中，使访问产生的数据持久化存储的一种技术

会话技术的分类

  • COOKIE技术
  • SESSION技术

COOKIE技术

COOKIE技术是神马

  • 将数据持久化存储在浏览器端，并且还能标识不同浏览器的一种技术

setcookie 与 $_COOKIE 变量之间的关系

注意点:

  • 只有当执行程序之前，$_COOKIE才会被PHP操作一次，将浏览器携带过来的所有COOKIE数据赋值给\$_COOKIE数组；
  • ( setcookie 函数间接操作 $_COOKIE ) setcookie函数与 \$_COOKIE没有直接的操作关系，setcookie函数只有一个作用，在本次接下来响应浏览器的过程中负责告诉浏览器将要设置一条什么样的COOKIE数据；
    问题:
  • ssetcookie后，什么时候才能通过$_COOKIE变量获得该数据？

在setcookie函数执行的本次请求之后的所有请求中才能获得该数据

COOKIE数据的相关属性设置

  • 有效期

不设置或设置为0表示默认情况，为一个会话周期( 默认情况一个会话周期为从新增该条COOKIE到浏览器关闭为止 )；

注意点:

通过设置一个过了期的时间，可以实现删除某条已有的COOKIE数据。

  • 有效路径

不设置或设置为空字符串表示默认情况( 默认情况表示当前程序所在目录及其子目录有效 )；

  • 有效域

不设置或设置为空字符串表示默认情况( 默认情况表示当前域名及其子域名有效 )；

  • 是否安全传输

不设置或设置为false表示默认情况( 默认情况表示http或https下都有效，如果将值设置为true,则表示只有在https下才有效（包括增删改查） )；

  • httponly

不设置或设置为false表示默认情况( 默认情况表示除了能够使用原程序脚本代码访问以外，还能够通过其他脚本语言访问到COOKIE数据。如果设置为true，则表示只允许原程序脚本代码进行操作 )；

COOKIE技术的局限性

  • COOKIE数据保存在浏览器端（用户机器上），不可控因素高，安全性相对比较低
  • 每次请求都会携带所有的COOKIE数据，如果数据量大，将会占用大量的带宽，降低网站加载效率

SESSION技术

SESSION技术是神马

  • 将数据持久化存储在服务器端的一门技术。（并且还要使数据能够区分不同的浏览器）

SESSION技术的原理

  • 保存SESSION数据的空间构建一个唯一的“口令”，这个“口令”会以COOKIE数据的形态存在 ( 所以说SESSION技术本质上是依赖于COOKIE技术的 )

SESSION会话数据区与$_SESSION的关系

session操作特点

  • ( 间接操作SESSION会话数据区 )操作$_SESSION变量并不会直接操作到SESSION会话数据区
  • 整个执行程序的过程中，只有两次操作到SESSION会话数据区的机会

第一次是当执行到session_start这句代码时，才会去找SESSION会话数据区，如果没有则开辟一个全新的SESSION会话数据区

第二次是当整个程序执行结束时，才会将$_SESSION中的所有元素数据一次性存储进SESSION会话数据区

销毁SESSION数据（区）

  • 通过session_destroy函数来清除当前会话所有的SESSION数据

SESSION（数据）的属性设置

通过配置php.ini相应的配置项来控制

  • 有效期

默认情况下也是一个会话周期 ( 一个会话周期表示生成这个COOKIE数据开始到浏览器关闭为止 ) 。

  • 有效路径

默认情况下是网站的根目录及其子目录下都有效

  • 有效域

当前域名及其子域名下有效

  • 是否安全传输

默认情况为http和https都能使用

  • httponly

这个属性对于SESSION无意义

注意点:改php.ini中的配置项的方法控制PHPSESSID的属性。所以项目中的操作策略是，什么时候需要，就什么时候在代码中去设置。

设置方法：通过session_set_cookie_params函数设置属性 ( 在session_start() 之前设置 )

SESSION的垃圾回收机制

PHP中存在一种针对SESSION会话数据区的垃圾回收机制，如果PHP发现，在一定的时间内，某个SESSION会话数据区没有被使用，则在下一次访问时，将会自动以一定的概率清除这个被认为是无效的垃圾SESSION会话数
